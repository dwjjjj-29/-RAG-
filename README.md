# 系统设计介绍

## 一句话定位

面向高校场景的混合检索增强生成（Hybrid RAG）智能问答系统，融合语义向量检索与关键词检索，为大语言模型提供准确的上下文，实现精准、可溯源的自动问答。

## 核心流程

```
用户提问 → 双路检索 → RRF融合 → 重排序 → LLM生成回答
```

- **双路检索**：语义向量检索（BGE + FAISS）+ 关键词检索（jieba + BM25），并行获取候选文档
- **RRF 融合**：倒数排序融合两路结果并去重
- **重排序**：交叉编码器对候选文档精细打分，取 Top-K
- **问答生成**：将精选文档作为上下文，调用大模型生成答案并附引用

## 技术选型

| 组件 | 技术 |
|------|------|
| 向量模型 | BAAI/bge-small-zh-v1.5 |
| 向量索引 | FAISS |
| 关键词检索 | BM25 + jieba |
| 融合算法 | RRF (Reciprocal Rank Fusion) |
| 重排模型 | BAAI/bge-reranker-base |
| 大语言模型 | DeepSeek v4（OpenAI 兼容接口） |
| 文档处理 | LangChain |
| Web 界面 | Gradio |

## 为什么用混合检索

纯向量检索会漏掉罕见词和精确术语，纯关键词检索无法理解同义表达。两路互补，RRF 融合两路优势，交叉编码器精排保证最终质量。
